# Security Policy

We take the security of our software seriously. If you discover a vulnerability, please report it to us using the steps below.

## Reporting a Vulnerability

To ensure we have enough time to investigate and fix the problem, please do not report security issues in public forums or issue trackers. 

Instead, choose one of the following private methods:
- **Email:** Send details to [Insert Security Email Address].
- **GitHub:** Use the [Report a vulnerability](https://github.com/github/form-templates/blob/main/SECURITY.md) option located under our repository's Security tab to start a private conversation with our team.

Please include the following details in your report:
* The exact steps to reproduce the issue.
* The version of the project you are using.
* Any code or tools required to test the issue.

## Supported Versions

We only provide security updates and patches for the most recent stable release. 

| Version | Supported |
| :--- | :--- |
| 2.x (Latest) | Yes |
| 1.x (Legacy) | No |

## Disclosure Policy

When you report a security issue, our team will:
1. **Acknowledge** your report within 48 hours.
2. **Review** the issue to confirm the vulnerability and scope.
3. **Work** on a fix and test it.
4. **Publish** a security advisory and release a patched version of the software.

We ask that you keep the vulnerability private until we have had time to release a fix to the public. Thank you for your help in keeping this project secure!
