# Architecture Decision Record

## Context
Most retail backtesting software tightly couples

 - strategies
 - execution
 - portfolio management
 - analytics

This makes experimentation difficult.

---

## Decision
QTLab will adopt a modular architecture where

 - strategies generate signals
 - execution fills orders
 - portfolio tracks positions
 - analytics evaluate results

Each subsystem is independently testable.

---

## Consequences

### Advantages

 - extensible
 - reusable
 - easier testing
 - easier live trading

### Tradeoffs

 - more classes
 - more interfaces
 - higher initial complexity
