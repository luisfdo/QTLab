# Project Architecture

## Decision

QTLab will adopt a layered architecture
where research components are isolated
from execution components.

- strategies generate signals
- execution fills orders
- portfolio tracks positions
- analytics evaluate results

Each subsystem is independently testable.


## Context

Retail backtest engines often mix

- strategies
- execution
- portfolio management
- analytics

This makes experimentation difficult.


## Consequences

### Advantages

Much greater long-term extensibility.

- extensible
- reusable
- easier testing
- easier live trading

### Tradeoffs

Higher upfront complexity.

- more classes
- more interfaces
- higher initial complexity
