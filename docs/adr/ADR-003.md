# Domain Models

## Decision

>Domain entities in QTLab SHALL be implemented using Pydantic models.

## Context

Domain entities have invariants that must always hold.

Examples:

- OHLC relationships
- Non-negative prices
- Timezone-aware timestamps
- Immutable state

## Rationale

Pydantic provides:

- validation
- serialization
- excellent error messages
- strong typing
- immutability
- future API compatibility

## Trade-offs

- Slight runtime overhead
- Additional dependency

## Status

**Accepted**
