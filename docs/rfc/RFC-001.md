# Domain Models

## Dataclass vs pydantic

| Principle                        | Dataclass            | Pydantic              |
|----------------------------------|----------------------|-----------------------|
| Correctness before performance   | ⚠️ Manual validation | ✅ Built-in validation |
| Make illegal states impossible   | ⚠️ __post_init__     | ✅ Validators          |
| Explicit is better than implicit | ✅                    | ✅                     |
| Immutable domain objects         | ✅                    | ✅ (frozen=True)       |
| Testability                      | ✅                    | ✅                     |
| Developer experience             | ⚠️ More boilerplate  | ✅ Excellent           |
| Error reporting                  | ⚠️ Manual            | ✅ Exceptional         |

To me, one row stands out:

>Make illegal states impossible.
